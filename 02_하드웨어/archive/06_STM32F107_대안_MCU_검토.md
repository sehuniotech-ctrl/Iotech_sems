# STM32F107 대안 MCU 검토

> 현재 문서는 후보 비교 이력입니다. 최종 선정 MCU는 `STM32G071CBT6`입니다.

작성일: 2026-04-02  
기준: 단상 지선차단기 / 계량칩 `BL0942` / 개발환경 `STM32CubeMX`

## 1. 결론 먼저

`STM32F107` 말고도 대안은 충분히 있다.  
이번 프로젝트 기준으로 보면 아래 3개가 가장 현실적이다.

1. `STM32F103C8T6`: 가장 저렴하고 가장 무난한 대안
2. `STM32G071CBT6`: 가격과 성능의 균형이 좋은 최신 대안
3. `STM32C071CBT6`: 최소 구성으로 갈 때 고려 가능한 대안

그리고 여유가 더 필요하면:

- `STM32G0B1CCT6`: 기능 여유가 큰 상위 대안
- `STM32F303CCT6`: 아날로그 기능이 많지만 이번 용도에는 다소 과함

## 2. 이번 프로젝트에서 MCU가 해야 하는 일

MCU 필수 역할:

- BL0942 통신
- RS-485 통신
- 릴레이 제어
- 상태 LED 제어
- 이벤트 로그 처리
- 계측값 가공
- 향후 서버/DCU용 프로토콜 처리

따라서 최소 조건은 아래다.

- USART 최소 2개
- GPIO 여유
- Flash / RAM 너무 작지 않을 것
- CubeMX 지원
- 양산 공급성 무난할 것

## 4. 후보별 해석

### 4.1 STM32F103C8T6

장점:

- 가격이 매우 좋다
- 재고가 많다
- 48핀이라 보드 크기 줄이기 좋다
- USART 3개라 `BL0942 + RS-485 + 서비스 UART` 구성이 가능하다
- 개발 자료가 많다
- 10년 longevity를 exact part 기준으로 바로 설명하기 좋다

주의:

- SRAM이 크지 않다
- 오래된 계열이라 최신 주변기능은 적다
- 대형 로그/복잡한 통신/기능 확장에는 여유가 적을 수 있다

판단:

- `가장 현실적인 저가 대안`

### 4.2 STM32G071CBT6

장점:

- 비교적 최신 계열
- USART 4개로 통신 여유가 좋다
- Flash 128KB / SRAM 36KB면 이번 용도에 충분하다
- 가격도 아주 비싸지 않다
- 48핀 패키지라 보드가 작아진다
- G0 계열 10년 longevity 프로그램 확인 가능

주의:

- F103보다 단가는 높다
- Cortex-M3가 아니라 M0+라 일부 개발자에게 익숙함 차이가 있을 수 있다
- exact part의 longevity 행은 발주 전에 한 번 더 직접 확인하는 편이 안전하다

판단:

- `가격/기능/공급성 균형이 가장 좋은 대안`

### 4.3 STM32C071CBT6

장점:

- 가격이 낮다
- 48핀으로 작다
- Flash 128KB라 코드 공간은 괜찮다
- CubeMX 지원
- C0 계열 10년 longevity 프로그램 확인 가능

주의:

- USART가 `2개`뿐이라 딱 맞춘 설계가 된다
- 재고가 적게 보이는 시점이 있어 공급 안정성은 재확인 필요
- 기능 여유가 작다
- exact part longevity는 발주 전에 다시 확인 권장

판단:

- `딱 필요한 것만 넣는 최소 원가형`으로는 가능
- 하지만 통신/디버그 여유가 적다

### 4.4 STM32G0B1CCT6

장점:

- USART 6개
- SRAM 144KB
- Flash 256KB
- 확장성 매우 좋음
- G0B1 계열 10년 longevity 프로그램 확인 가능

주의:

- 이번 프로젝트에는 다소 과함
- 가격이 올라간다
- exact part longevity는 발주 전에 다시 확인 권장

판단:

- `향후 기능 확장까지 고려하면 좋은 상위 대안`

### 4.5 STM32F303CCT6

장점:

- M4F 코어
- 아날로그 기능이 매우 좋다
- 성능 여유가 많다
- exact part 기준 10년 longevity 확인 가능

주의:

- 단가가 높다
- 이번 프로젝트는 BL0942를 별도 계량칩으로 쓰기 때문에 F303의 강점이 크게 살아나지 않는다

판단:

- `이번 용도에는 과한 편`

## 5. 그래서 어떤 칩이 좋은가

### 선택지 1. 개발 안정성 우선

- `STM32F107VCT6`

이 경우:

- 핀 많음
- 기능 여유 큼
- 개발은 편함
- 대신 보드가 크고 MCU가 과함

### 선택지 2. 가장 추천

- `STM32G071CBT6`

이유:

- 48핀
- USART 4개
- Flash/RAM 적당
- 최신 계열
- 가격도 아직 수용 가능
- 10년 longevity 계열로 관리 가능

### 선택지 3. 가장 저렴한 실무 대안

- `STM32F103C8T6`

이유:

- 재고 좋음
- 가격 좋음
- 1차 제품에는 충분
- 10년 longevity를 exact part 기준으로 설명 가능
- 단, 메모리/여유는 G071보다 불리

## 6. 내 추천

이번 프로젝트 기준 추천 순서는 아래다.

1. `STM32G071CBT6`
2. `STM32F103C8T6`
3. `STM32F107VCT6`

정리하면:

- `보드 작게 + 기능 충분 + CubeMX + 통신 여유`: `STM32G071CBT6`
- `최저가 중심`: `STM32F103C8T6`
- `기존 방향 유지 + 핀 여유`: `STM32F107VCT6`

## 7. Longevity 정리

ST는 공식 `Product Longevity` 프로그램을 운영하고, STM32는 `rolling 10-year commitment` 방식으로 관리된다.

이번 추천 대안들에 대한 확인 결과는 아래와 같다.

| MCU | Longevity | 시작일 | 확인 수준 | 근거 |
|---|---|---|---|---|
| `STM32F103C8T6` | `10년` | `2025-01-01` | 직접 확인 | ST 공식 제품 페이지 `Quality and Reliability` 표 |
| `STM32G071CBT6` | `10년` | `2025-01-01` | 계열 기준 확인 | ST 공식 `STM32G071KB` 제품 페이지에서 `Longevity Commitment 10`, 시작일 `2025-01-01` 확인 |
| `STM32C071CBT6` | `10년` | `2025-02-19` | 계열 기준 확인 | ST 공식 `STM32C071KB` 제품 페이지에서 `Longevity Commitment 10`, 시작일 `2025-02-19` 확인 |
| `STM32G0B1CCT6` | `10년` | `2025-01-01` | 계열 기준 확인 | ST 공식 G0B1 계열 제품 페이지에서 `Longevity Commitment 10`, 시작일 `2025-01-01` 확인 |
| `STM32F303CCT6` | `10년` | `2025-01-01` | 직접 확인 | ST 공식 제품 페이지 `Quality and Reliability` 표 |

설명:

- `직접 확인`은 exact part number가 포함된 제품 페이지 표에서 longevity 값이 직접 보인 경우다.
- `계열 기준 확인`은 exact part number는 맞지만 현재 수집 가능한 ST 페이지 구조상 longevity 열이 직접 추출되지 않아, 같은 family의 다른 package/pin-count 제품을 근거로 판단한 경우다.

실무 판단:

- `STM32F103C8T6`, `STM32F303CCT6`는 10년 보장을 문서에 자신 있게 넣을 수 있다.
- `STM32G071CBT6`, `STM32C071CBT6`, `STM32G0B1CCT6`도 10년 계열로 보는 것이 합리적이지만, 최종 발주 전에는 exact part 기준으로 다시 직접 확인하는 것이 좋다.

## 9. 메모

- 가격과 재고는 시점에 따라 자주 바뀐다.
- `STM32C071CBT6`는 이번 조회 시점에서 LCSC 재고가 매우 적게 보였으므로 발주 전에 재확인이 꼭 필요하다.
- 이번 프로젝트에서 `BL0942`를 별도 계량칩으로 쓸 계획이면, MCU는 `통신/제어/저장` 중심으로 선택하는 것이 맞다.
- `STM32G071CBT6`, `STM32C071CBT6`, `STM32G0B1CCT6`의 longevity는 현재 수집한 ST 페이지 구조상 exact part의 수치가 동일 페이지에서 텍스트로 깨끗하게 추출되지 않아, 같은 family 제품 페이지를 근거로 판단했다. 이 부분은 발주 전 다시 확인 권장이다.
