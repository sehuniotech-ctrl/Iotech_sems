# STM32F107 + BL0942 권장 핀맵 초안

> 현재 문서는 초기 후보 검토 이력입니다. 현행 선정 조합은 `STM32G071CBT6 + BL0942`입니다.

작성일: 2026-04-02  
대상 MCU: `STM32F107VCT6`  
대상 계량칩: `BL0942`  
기준: 단상 지선차단기 1차 제품

## 1. 이 문서의 기준

이번 초안은 아래 기준으로 잡았다.

- `STM32CubeMX`에서 바로 시작하기 쉬울 것
- `기본 핀 기능` 위주로 사용해서 remap 의존을 줄일 것
- `BL0942 + RS-485 + 릴레이 + 디버그`를 우선할 것
- `USB / Ethernet / CAN / ZCT`는 1차 범위에서 제외할 것

즉, 이번 초안은 `1차 제품을 빨리 시작하기 위한 핀맵`이다.

## 2. 핀 분류 기준

### 2.1 필수

회로도에서 사실상 반드시 고정해야 하는 핀

### 2.2 권장

1차 제품에서는 쓰는 것이 좋거나, 향후 디버깅/확장 때문에 남겨두는 것이 좋은 핀

## 3. 필수 핀

### 3.1 시스템 필수 핀

| 구분 | 핀 | 용도 | 비고 |
|---|---|---|---|
| 전원 | `VDD`, `VSS` | 디지털 전원/그라운드 | 전부 연결 필수 |
| 아날로그 전원 | `VDDA`, `VSSA` | 아날로그 전원/그라운드 | 반드시 안정적으로 처리 |
| 리셋 | `NRST` | 외부 리셋 | 디버그/복구 위해 필수 |
| 부트 | `BOOT0` | 부트 모드 선택 | 풀다운 권장, 점퍼/패드 권장 |
| 디버그 | `PA13` | `SWDIO` | 필수 |
| 디버그 | `PA14` | `SWCLK` | 필수 |
| 클럭 | `PD0` | `OSC_IN` | HSE 사용 권장 |
| 클럭 | `PD1` | `OSC_OUT` | HSE 사용 권장 |

### 3.2 프로젝트 필수 기능 핀

| 구분 | 핀 | 용도 | 선정 이유 |
|---|---|---|---|
| BL0942 UART | `PA2` | `USART2_TX` | 기본 핀, CubeMX 시작 쉬움 |
| BL0942 UART | `PA3` | `USART2_RX` | 기본 핀, CubeMX 시작 쉬움 |
| RS-485 UART | `PB10` | `USART3_TX` | 통신 전용 UART로 분리 |
| RS-485 UART | `PB11` | `USART3_RX` | 통신 전용 UART로 분리 |
| RS-485 방향제어 | `PB12` | `DE/RE GPIO` | USART3 주변에 배치하기 좋음 |
| 릴레이 제어 | `PC6` | Relay Drive | 단순 GPIO 출력 |
| 릴레이 피드백 | `PC7` | Relay Feedback | 상태 확인용 권장, 실사용 기준 필수급 |
| 상태 LED | `PC8` | RUN LED | 동작 확인 |
| 상태 LED | `PC9` | TRIP/FAULT LED | 장애 확인 |

## 4. 권장 핀

### 4.1 서비스/디버그 권장 핀

| 구분 | 핀 | 용도 | 비고 |
|---|---|---|---|
| 서비스 UART | `PA9` | `USART1_TX` | 부트로더/로그용으로 매우 유용 |
| 서비스 UART | `PA10` | `USART1_RX` | 부트로더/로그용으로 매우 유용 |
| RTC 클럭 | `PC14` | `OSC32_IN` | RTC 쓰면 사용 |
| RTC 클럭 | `PC15` | `OSC32_OUT` | RTC 쓰면 사용 |

추천 이유:

- `PA9/PA10`은 시스템 메모리 부트로더 쪽에서도 활용성이 좋아서 비워두면 나중에 살리기 편하다.
- `PC14/PC15`는 RTC가 필요해질 가능성이 있으면 미리 살려두는 편이 좋다.

### 4.2 BL0942 보조 신호 권장 핀

| 구분 | 핀 | 용도 | 비고 |
|---|---|---|---|
| BL0942 보조입력 | `PB0` | `ZX` 또는 이벤트 입력 | 영점 통과/보조 인터럽트용 |
| BL0942 보조입력 | `PB1` | `CF` 또는 펄스 입력 | 에너지 펄스 확인용 |

추천 이유:

- UART만으로도 기본 구현은 가능하지만, `ZX/CF`를 받아두면 검증과 디버깅이 쉬워진다.

### 4.3 향후 확장용 권장 핀

| 구분 | 핀 | 용도 | 비고 |
|---|---|---|---|
| 예비 GPIO/ADC | `PC0` | 확장 입력 | 온도/ZCT/아날로그 확장 후보 |
| 예비 GPIO/ADC | `PC1` | 확장 입력 | 온도/ZCT/아날로그 확장 후보 |
| 예비 GPIO/ADC | `PC2` | 확장 입력 | 향후 센서 확장 |
| 예비 GPIO/ADC | `PC3` | 확장 입력 | 향후 센서 확장 |

## 5. 이번 권장 배치의 핵심 의도

### 5.1 BL0942는 `USART2`

이유:

- `PA2/PA3`는 기본 UART 핀이라 다루기 쉽다
- `USART1`을 서비스/부트로더용으로 남길 수 있다
- 펌웨어 개발 중 디버그 경로를 분리하기 좋다

### 5.2 RS-485는 `USART3`

이유:

- 외부 통신과 계량칩 통신을 분리할 수 있다
- `PB10/PB11` 조합은 회로도에서 이해하기 쉽다
- `PB12`를 DE/RE로 붙이면 배선도 정리하기 좋다

### 5.3 릴레이와 LED는 `PC6~PC9`

이유:

- 연속된 GPIO로 묶어서 회로와 코드 가독성이 좋다
- 통신/디버그 핀과 분리되어 충돌이 적다

## 7. HW팀 전달 시 같이 말해야 할 것

- 이번 1차는 `기본 remap 최소화`로 간다
- `USART2 = BL0942`, `USART3 = RS-485`로 우선 설계
- `USART1`은 서비스/로그/복구용으로 비워두는 방향
- `SWD`, `NRST`, `BOOT0`, `HSE`는 반드시 살린다
- `PC14/PC15`는 RTC 필요 여부 보고 최종 결정
- `PB0/PB1`은 BL0942 `ZX/CF` 검증용으로 연결 권장
