# HW팀 전달용 선정칩 기준 핀 기능 정리

작성일: 2026-04-10

## 1. 목적

본 문서는 현재 선정된 부품 조합인 `STM32L053R8T6 + ATM90E26-YU` 기준으로,

- 전체 핀 기능을 어떻게 사용할지
- 어떤 핀은 필수로 회로에 반영해야 하는지
- 어떤 핀은 권장 또는 확장용으로 남겨둘지

를 HW팀 전달용으로 표 중심으로 정리한 문서다.

## 2. 선정 기준 칩

| 구분 | 부품명 | 패키지 | 비고 |
|---|---|---|---|
| MCU | `STM32L053R8T6` | `LQFP64` | 현재 기준 MCU |
| 계량칩 | `ATM90E26-YU` | `SSOP-28` | 현재 기준 계량칩 |

## 4. MCU 쪽 핀 분류

### 4.1 분류 기준

| 분류 | 의미 |
|---|---|
| 필수 | 1차 제품에서 반드시 회로 반영해야 하는 핀 |
| 권장 | 1차 제품에서 가능하면 반영하는 것이 좋은 핀 |
| 확장 | 2차 기능 또는 디버깅/생산 대응을 위해 열어두면 좋은 핀 |

### 4.2 MCU 필수 핀

| 구분 | MCU 핀 | 기능 | 연결 대상 | 필수 여부 | 비고 |
|---|---|---|---|---|---|

| 계량칩 SPI | `PA5` | `SPI1_SCK` | ATM90 `SCLK` | 필수 | ATM90 통신 |

## 7. HW팀 우선 반영 요청 항목

### 7.1 반드시 반영할 항목

| 구분 | 내용 |
|---|---|
| MCU 기본 | 전원, GND, NRST, SWDIO, SWCLK |
| MCU 통신 | `USART2 TX/RX + DE/RE` |
| MCU 제어 | 릴레이 구동 GPIO |
| MCU ↔ ATM90 | `SPI1 + CS + Reset` |
| ATM90 기본 | 전원, GND, Vref, OSCI/OSCO, I1P/I1N, VP/VN |
| ATM90 설정 | `USEL`, `MMD1`, `MMD0`, Reserved 핀 데이터시트 준수 |

### 7.2 가능하면 반영할 항목

| 구분 | 내용 |
|---|---|
| 릴레이 피드백 | 릴레이 실제 상태 확인 입력 |
| 서비스 UART | 생산/디버그용 `USART1` |
| 상태 LED | RUN / FAULT 2개 |
| ATM90 보조핀 | `IRQ`, `WarnOut`, `ZX`, `CF1`, `CF2` |
| I2 채널 | `I2P/I2N` 확장 여지 |

## 8. 주의사항

- 본 문서는 HW팀 전달용 기능-핀 기준 문서다.
- 실제 최종 AF 매핑과 핀 충돌 여부는 `STM32CubeMX` 및 데이터시트에서 최종 확인이 필요하다.
- 현재 기준은 `SPI 기반 ATM90 연결 + USART2 기반 RS-485 + GPIO 릴레이 제어` 구조다.
- 1차 제품에서는 필수 핀 우선, 권장 핀은 보드 여유와 생산성 기준으로 반영 여부를 판단한다.

## 9. 결론

현재 선정 조합인 `STM32L053R8T6 + ATM90E26-YU` 기준으로,

- 필수 회로는 `MCU 기본 + SPI 계량칩 + RS-485 + 릴레이 제어`
- 권장 회로는 `릴레이 피드백 + 서비스 UART + 상태 LED + ATM90 보조 출력`

으로 정리할 수 있다.

HW 설계는 우선 필수 핀을 고정하고, 보드 공간과 생산 검증 관점에서 권장 핀을 최대한 반영하는 방향이 적절하다.
