# SEMS 4개 장비 구성 및 역할 정리

작성일: 2026-04-16  
기준 대상:

- `SL` = Smart Load(기존 지선차단기)
- `DCU`
- `3상점검장치`
- `MQTT 서버`

## 1. 목적

본 문서는 SEMS를 구성하는 4개 장비를 기준으로 각 장비의 역할, 데이터 흐름, 책임 범위, 제안 포인트를 한 번에 정리하기 위한 문서이다.

## 2. 전체 구조

기준 구조는 아래와 같이 본다.

`MQTT 서버 - 3상점검장치 - (PLC) - DCU - SL`

## 3. 장비별 역할

### 3.1 SL

- 분기회로 말단 장치
- 전압, 전류, 전력, 전력량 등 직접 계측
- 과전류/과전압/대기전력 등 현장 판단
- 차단 및 복구 동작 수행
- 상태/이벤트/계측 데이터를 DCU에 전달

핵심 정의:

`SL은 SEMS의 말단 실행형 현장단말이다.`

### 3.2 DCU

- 여러 SL을 수집하는 현장 집선 장치
- 폴링 주기 관리
- 장치 식별 및 통신 상태 관리
- 상위 장치로 데이터 전달
- 필요 시 SL 제어 명령 전달

핵심 정의:

`DCU는 현장 장치들을 연결하는 집선/중계 제어 장치다.`

### 3.3 3상점검장치

- 메인 배전반 측 기준 장치
- 현장 상위 게이트웨이 역할
- 전체 전원 상태 및 기준 데이터 확보
- DCU 또는 하위 구조와 연계
- 상위 MQTT 서버로 가는 중간 계층 역할

핵심 정의:

`3상점검장치는 SEMS 현장 상위 게이트웨이이자 기준 계측 장치다.`

### 3.4 MQTT 서버

- 상위 플랫폼/서버 계층
- 데이터 수집, 저장, 분석
- 대시보드, 이력, 알람, 운영 정책 처리
- 장치별 상태 및 이벤트 통합 관리
- 향후 에너지절감/운영 최적화 로직 수행

핵심 정의:

`MQTT 서버는 SEMS의 상위 운영 및 분석 플랫폼이다.`

## 4. 데이터 흐름

### 4.1 상향 데이터

1. SL이 계측값과 이벤트를 생성
2. DCU가 여러 SL의 데이터를 수집
3. 3상점검장치가 현장 기준 정보와 함께 상위 연계
4. MQTT 서버가 전체 데이터를 저장 및 분석

### 4.2 하향 제어

1. MQTT 서버 또는 상위 운영 정책이 제어 필요 판단
2. 3상점검장치/DCU를 통해 대상 장치 지정
3. DCU가 SL로 제어 명령 전달
4. SL이 차단/복구/상태 응답 수행

## 5. 장비별 핵심 데이터

| 장비 | 입력 | 출력 | 핵심 가치 |
|---|---|---|---|
| SL | 전력 신호, 현장 조건, 제어 명령 | 계측값, 이벤트, 차단 상태 | 현장 말단 계측과 즉시 차단 |
| DCU | SL 다수 데이터, 상위 명령 | 집선 데이터, 하위 제어 | 현장 통신 허브 |
| 3상점검장치 | 메인 배전반 정보, DCU 정보 | 상위 연계 데이터 | 기준 계측 + 게이트웨이 |
| MQTT 서버 | 현장 전체 데이터 | 대시보드, 분석, 알람, 운영 정책 | 통합 운영 플랫폼 |

## 6. 제안 포인트

### 6.1 SL 제안 포인트

- 말단 회로 직접 계측
- 자체 판단 및 차단
- 상위 분석용 원천 데이터 제공

### 6.2 DCU 제안 포인트

- 다수 SL 연결 및 관리
- 제어 명령 전달 허브
- 현장 통신 안정성 확보

### 6.3 3상점검장치 제안 포인트

- 메인 배전반 기준 계측
- 현장 상위 게이트웨이
- 상하위 시스템 연결

### 6.4 MQTT 서버 제안 포인트

- 데이터 저장/분석/알람 통합
- 다현장/멀티사이트 확장 가능
- 향후 에너지절감 운영 로직 확장 가능

## 7. 역할 구분에서 중요한 점

- `SL`은 단순 센서가 아니다.
- `DCU`는 단순 중계기가 아니라 장치 운영 허브다.
- `3상점검장치`는 메인 배전반 기준 장치이자 게이트웨이다.
- `MQTT 서버`는 단순 저장소가 아니라 운영 플랫폼이다.

## 8. 대외 설명용 한 문장

`SEMS는 SL이 현장 분기회로를 직접 계측하고 차단을 수행하며, DCU와 3상점검장치가 이를 집선·연계하고, MQTT 서버가 전체 데이터를 분석·운영하는 4계층 구조의 에너지관리 시스템이다.`
