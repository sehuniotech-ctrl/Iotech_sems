# 2026-04-03 SEMS 회의 정리

작성일: 2026-04-03  

## 1. 시스템 구조 및 방향

### 1.1 전체 구조

- `MQTT서버 - 3상 점검장치 - (PLC) - DCU - 지선차단기`
- 3상 점검장치는 메인 배전반 장치로 본다
- PLC 통신은 가능하다고 보고 있다
- 실제 구현 가능 여부와 구체 방식은 추후 상세 검토

### 1.2 3상 점검장치 역할

- 현재 3상 점검장치는 `게이트웨이` 역할이다
- 우리가 어떤 데이터를 보내야 하는지는 WESYNC 측에 전달해야 한다
- 큰 방향은 전달되어 있으나, 세부 데이터 항목은 추가 전달 필요

## 2. 지선차단기 관련 방향


### 2.2 정보 전달 방식

- 각 지선차단기는 정보를 취합하지 않는다
- 각 지선차단기가 계량한 정보를 직접 전송한다

### 2.3 수량/구조 관련

- IP를 부여하는 구조에서 최대 `254개` 수준 언급
- slave 최대 개수는 `40개가량`으로 정리

### 2.4 개발 범위

- 현재는 `삼상 진행 안 함`
- `단상만 진행`
- 삼상은 `TBD`

## 3. 프로젝트 운영

### 3.1 PM

- 프로젝트 PM: `김덕회 팀장`

### 3.2 PM 역할

- 프로젝트 전체 일정 관리
- 역할 분담 및 조율
- 진행 상황 점검
- 이슈 취합 및 관리
- 필요한 자료 및 커뮤니케이션 정리

### 3.3 참고 자료

- `김상호 이사님 카톡 파일` 참고

## 4. 지선차단기 기능 관련

### 4.1 기능 추가 방향
- 지선차단기에 차단 기능 추가 필요
- 과전류, 과전압 차단은 지선차단기 자체가 판단해서 수행


### 4.2 복구 방식

- 차단 후 `복구를 어떻게 할지` 방법 검토 필요

검토 필요 항목:

- 자동 복구
- 수동 복구
- 원격 복구
- 복구 조건 및 안전 정책

## 5. 개발 범위 및 판매 타겟팅

### 5.1 개발 범위

- 1차 개발 범위
- 2차 개발 범위

### 5.2 판매 타겟팅

- 판매 타겟을 어디로 잡을지 정리 필요

즉, 아래 2개를 같이 정리해야 한다.

- 무엇을 만들 것인지
- 누구에게 판매할 것인지

## 6. 레퍼런스 및 데이터

- `한국 에너지산업` 쪽 기술 및 데이터 확보 필요
- 영상장치 / 절감장치 레퍼런스 존재
- 실제 누적 데이터 확인 필요

정리:

- 단순 설명보다 실제 데이터와 사례 확보가 중요함
- 향후 의사결정을 위해 정리된 자료가 필요함

## 7. 영상분석기

- 영상분석기는 `추후 추가 아이템`
- 1차 개발 범위에는 포함하지 않음
- 향후 확장 아이템으로 별도 관리

## 8. 1차 개발 설치 위치

- 1차 개발에서는 메인 배전반에 설치

## 9. 연결 업체 관련

- 연결 업체에 바로 연락하는 것은 반대 의견
- 현재는 내부 정리와 범위 확정이 우선

## 10. 열화상 카메라 관련

### 10.1 현재 상황

- 열화상 카메라는 부스바 온도 감시에 현재 많이 사용됨
- 현재 검토 중인 열화상 카메라도 부스바 온도용으로 개발된 장비

### 10.2 협의 및 일정

- 협의에 `20일 이상` 필요
- 인터넷 환경이 되면 `4~5군데` 시범 설치 검토
- 목표 시점은 `5월`

### 10.3 베트남 관련

- 베트남에 우리 제품을 보여줌
- 베트남에서 오신 분은 우리나라 `300군데` 정도를 관리 중인 분
- 요구사항:
  - 전압 보고 싶음
  - 전류 보고 싶음
  - 부스바 온도상승 보고 싶음

### 10.4 인증

- 베트남용 인증이 따로 있음
- 판매 대수가 크지 않으면 한국 KC 인증만으로 큰 문제는 없다는 의견
- 큰 숫자로 갈 경우 세관 통과를 위해 별도 인증 필요

### 10.5 기타 참고 사항

- 열화상카메라 앱 Guarding vision은 우리 사무실에서 계속 확인하지 않게 함

## 11. 3상 점검장치 관련

- 지금 거기서 쓸 수 있는 것은 `3상 점검장치에 ZCT를 써서 계측항목 계측 가능`한 구조
- 3상 점검장치에 `PoE` 있음

## 12. WESYNC 개발 진행 상황

- 시운전 스케줄은 `4월` 기준 맞추는 방향
- 따라서 3상 점검장치 스케줄은 가능하다고 판단

## 13. 디바이스 서버 - 카메라 관련

### 13.1 현재 구조

- 디바이스 서버는 시리얼 사용 중
- `485`, `232` 변경 가능

### 13.2 현재 이슈

- 부스바 온도 체크 필요
- `G-type 계량기` 존재
- 인터넷과 카메라 간 거리가 멀다
- 인터넷과 카메라 간 통신을 `PLC`로 가져갈 수 있는지 검토 필요

### 13.3 디바이스 서버 기능 방향

- 디바이스 서버의 마스터 기능 검토 필요
- 구조 방향:
  - 마스터에서 slave로 request를 보내는 구조가 아니라
  - 서버에서 마스터로 request를 보내는 구조 검토

### 13.4 연결 기능

- 카메라 연결
- 계량기 연결
- 필요 시 계량기 별도 구매 및 설치

## 14. 이번 회의 기준 정리

- 3상 점검장치는 현재 게이트웨이 역할로 본다
- 단상만 우선 진행하고 삼상은 TBD로 둔다
- 지선차단기는 각 장치가 자기 정보를 직접 보낸다
- PM은 김팀장님이 맡는다
- 과전류/과전압은 지선차단기 자체 판단 구조로 간다
- 복구 방식은 별도 검토 필요
- 영상분석기와 열화상 카메라는 후속 확장 아이템으로 본다
- 1차/2차 개발 범위와 판매 타겟팅을 같이 정리해야 한다
- WESYNC에는 세부 데이터 항목을 추가로 전달해야 한다
