# STM32L053R8T6 ATM90E26 펌웨어 착수 준비

작성일: 2026-04-08

## 1. 기준

- MCU: `STM32L053R8T6`
- 계량칩: `ATM90E26-YU`
- 통신: `RS-485`
- 제품 명칭: `Smart Load`

## 2. 현재 목표

- ATM90E26 초기화 및 통신 확인
- 전압/전류/유효전력/무효전력/피상전력/역률/주파수 읽기
- RS-485 polling 응답 구조 유지
- 릴레이 차단 및 복구 로직 유지
- 보호 판단은 MCU 로직으로 유지

## 3. 인터페이스 전략

ATM90E26은 `UART` 또는 `SPI`를 사용할 수 있으므로, 펌웨어 구조는 transport abstraction으로 설계한다.

현재 권장 순서:

1. HW팀에서 인터페이스 방식 확정
2. 인터페이스 확정 전까지는 공통 driver layer 유지
3. 실제 HAL 연동은 `atm90e26_transport` 계층에서 처리

## 4. 펌웨어 모듈

- `atm90e26`
  - 레지스터 read/write
  - 초기화 시퀀스
  - 계측값 읽기
- `atm90e26_transport`
  - SPI/UART transport 분리
- `relay_control`
  - 릴레이 제어
- `protection`
  - 과전류/과전압/대기전력 판단
- `rs485_link`
  - DCU polling 응답
- `device_app`
  - 전체 앱 흐름

## 5. 현재 읽을 계측값

- 전압 RMS
- L line 전류 RMS
- 유효전력
- 무효전력
- 피상전력
- 역률
- 주파수
- 유효전력량
- 무효전력량

## 6. 이후 확장 가능한 항목

- 위상각
- 전압 sag 이벤트
- 정/역전력 방향 상태
- 무부하 상태
- `IRQ`, `WarnOut`, `ZX`, `CF1`, `CF2` 활용

## 7. CubeMX에서 다음으로 필요한 것

- ATM90E26 인터페이스 선택
  - `UART`면 `USART1`
  - `SPI`면 `SPI1` 권장
- `USART2`는 `RS-485` 유지
- `DE/RE`, `Relay`, `LED` GPIO 유지
- 필요 시 아래 입력 핀 추가
  - `IRQ`
  - `WarnOut`
  - `ZX`
  - `CF1`
  - `CF2`
