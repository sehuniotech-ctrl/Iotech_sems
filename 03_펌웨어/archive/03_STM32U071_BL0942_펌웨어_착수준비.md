# STM32U071 BL0942 펌웨어 착수 준비

작성일: 2026-04-06

## 1. 기준

- MCU: `STM32U071`
- 계량칩: `BL0942`
- 통신: `RS-485`
- 제품 범위: 단상 지선차단기 1차 제품

## 2. 펌웨어 목표

- BL0942 계측값을 주기적으로 읽는다.
- 지선차단기 계측값과 MCU 계산값을 RS-485로 전송한다.
- 과전류 등 보호 조건 발생 시 릴레이를 차단한다.
- 대기전력 차단 후 스케줄 복구, 원격 복구를 수행한다.
- 명령 결과는 push가 아니라 DCU polling 구조에 맞춰 동작한다.

## 3. 권장 소프트웨어 구조

- `bl0942`
  - BL0942 프레임 수신
  - raw 값 파싱
  - 계측값 변환
- `relay_control`
  - 릴레이 ON/OFF 제어
  - 릴레이 명령 이력 관리
- `protection`
  - 과전류 기준 비교
  - 보호 차단 수행
- `standby_recovery`
  - 대기전력 차단 판정
  - 스케줄 복구
  - 원격 복구
- `rs485_link`
  - DCU 명령 수신
  - polling 응답 프레임 생성
- `device_app`
  - 전체 상태 흐름 관리

## 4. CubeMX 시작 권장안

- Debug: SWD
- Clock: 내부 오실레이터 기반으로 먼저 bring-up 후 필요 시 외부 클럭 검토
- USART1: BL0942
- USART2: RS-485
- GPIO:
  - Relay control
  - RS-485 DE/RE
  - Status LED
- TIM:
  - 1ms tick 또는 주기성 task scheduling 용도

## 5. 주기성 task 예시

- 10ms
  - UART state machine
  - command parser
- 100ms
  - BL0942 frame decode
  - relay feedback check
- 500ms
  - protection evaluation
  - standby power evaluation
- 1s
  - measurement snapshot update
  - event timeout management

## 6. 주요 데이터 항목

- 직접 계측
  - 전압
  - 전류
  - 주파수
  - 유효전력
  - 누적 전력량
  - 과전류 판단용 fast RMS
- MCU 계산
  - 피상전력
  - 역률
- 이벤트
  - relay_on
  - relay_off
  - standby_cutoff
  - overcurrent_trip

## 7. 복구 정책 반영 포인트

- 수동 복구는 적용하지 않는다.
- 보호회로 차단 후 복구는 서버 명령 기반으로만 수행한다.
- 대기전력 차단 후 복구는 스케줄 복구와 원격 복구를 적용한다.
- 지선차단기는 명령 후 즉시 상태를 push 하지 않는다.
- DCU polling 요청에 대해 현재 계측값과 명령 반영 여부를 응답한다.

## 8. 착수 순서

1. STM32U071 IOC 생성
2. UART 2채널과 GPIO bring-up
3. BL0942 수신 프레임 확인
4. 계측값 변환 검증
5. RS-485 polling 응답 프레임 작성
6. 릴레이 제어 추가
7. 보호 차단 로직 추가
8. 대기전력 차단 및 복구 로직 추가
9. 이벤트/로그 정리

## 9. 현재 가정

- STM32U071의 exact package와 최종 핀맵은 회로도 확정 후 맞춘다.
- BL0942는 UART 기반으로 연결된다고 가정한다.
- 릴레이 차단 시 MCU 전원과 통신은 유지된다고 가정한다.
