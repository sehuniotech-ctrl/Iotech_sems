# Smart Load 일정 및 진행률 기준

작성일: 2026-04-09
기준 칩: `STM32L053R8T6 + ATM90E26-YU`
기준 방향: `LCU 보드 우선 데모 -> HW 설계/제작 -> 최종 보드 이식 -> 내부 데모`

## 1. 일정 운영 기준

- 오늘부터의 일정은 `데모 우선` 기준으로 운영한다.
- 계량칩 데모는 `LCU 보드 사용 가능 여부`를 먼저 확인한다.
- `LCU 보드 사용 가능` 시 `LCU 보드 + ATM90E26-YU`로 선행 검증한다.
- `LCU 보드 사용 불가` 시 `STM Eval Board + ATM90E26 SSOP 보드`로 즉시 전환한다.
- HW팀에는 기능 및 핀 기준을 이미 전달한 상태로 보고, 회로 설계와 펌웨어 선행 개발을 병행한다.

## 4. 실행 순서

1. `LCU 보드 사용 가능 여부`를 먼저 판정한다.
2. 가능하면 `LCU 보드` 기준으로 ATM90E26 데모 테스트를 진행한다.
3. 불가하면 즉시 `STM Eval Board + ATM90E26 SSOP 보드`로 전환한다.
4. ATM90E26 기본 계측값 read가 확인되면 `RS-485 polling`과 `릴레이 제어`를 붙인다.
5. 데모 코드가 안정화되면 `STM32L053R8T6 + ATM90E26-YU` 최종 보드 기준으로 이식한다.
6. 최종 보드 기준 코드가 올라가면 `MQTT 서버 연동`과 `측정 spec 확정`을 진행한다.
7. 기능 테스트 후 내부 데모로 마무리한다.
