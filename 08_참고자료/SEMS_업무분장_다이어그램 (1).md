## 2. 현재 기준 한 줄 정리

`Smart Load`는 SEMS 전체 시스템에서 `현장 말단의 계측 + 자체 판단 + 차단 + DCU 연동 + 서버용 원천데이터 생성`을 담당하는 장치로 정의한다.

## 3. 폴더 학습 결과 요약

### 3.1 현재 내부 문서에서 읽히는 공통 방향

- 대외 명칭은 `지선차단기`보다 `Smart Load`를 우선 사용한다.
- SEMS는 단순 계측기가 아니라 `에너지 절감 효과를 만들어내는 시스템`으로 해석한다.
- 전체 구조는 `MQTT 서버 - 3상 점검장치 - (PLC) - DCU - Smart Load` 방향으로 정리되어 있다.
- 각 Smart Load는 자기 데이터를 직접 계측하고 직접 전송한다.
- 과전류/과전압 차단은 Smart Load 자체 판단 구조로 간다.
- 1차 개발 범위는 `단상 우선`이다.
- `복구 정책`, `WESYNC 전달 데이터 항목`, `판매 타겟`, `1차/2차 범위`는 아직 PM 관점 최종 정리가 더 필요하다.

### 3.2 내부 문서상 기준 충돌과 해석

현재 문서들에는 아래 두 흐름이 같이 존재한다.

1. 과거 기준
- `STM32U073C8T6 + BL0942`
- 1차 최소 기능 시제품 중심

2. 현재 통합 기준
- `STM32L053R8T6 + ATM90E26-YU`
- 계측 항목 확장성과 향후 Smart Load 고도화까지 고려

따라서 본 문서에서는 아래처럼 해석한다.

- `BL0942 + U073` 문서는 `1차 축소형/과거 검토 기준`
- `ATM90E26-YU + L053` 문서는 `현재 기준안`

즉, 제안서와 대외 설명은 `현재 기준안`을 메인으로 쓰고, 필요 시 `축소형 대안`으로 `BL0942` 기반을 백업 시나리오로 둔다.

## 4. SEMS 시장 조사

### 4.1 시장 해석

SEMS 시장은 단순 계측기 판매 시장이 아니라 아래 3개가 결합된 시장으로 보는 것이 맞다.

- 현장 전력 분기 단위 계측
- 이상 징후 감지 및 보호
- 상위 플랫폼 기반 분석, 경보, 절감 운영

즉 시장은 `스마트 분기회로 모니터링 + 에너지관리 + 상태감시`가 합쳐진 형태로 움직이고 있다.

### 4.2 국내 수요 관점

국내에서는 에너지관리시스템 도입 자체를 정부가 지원하는 구조가 이미 존재한다.

- 한국에너지공단은 `산업계 에너지관리시스템 보급 지원사업`을 통해 중소·중견 사업장의 `계측·제어 인프라 및 모니터링 시스템 구축`과 `에너지절감 활동 지원`을 명시하고 있다.
- 이는 현장 계측 단말, 제어 인프라, 모니터링 시스템이 함께 묶여야 실제 도입사업이 성립한다는 뜻이다.

해석:

- `Smart Load`는 단품 차단기보다 `SEMS 구축용 현장단말` 포지션으로 설명하는 것이 더 유리하다.
- 영업 포인트는 `차단기 1개 판매`보다 `현장 데이터 인프라 구축 + 절감 운영 기반`이 되어야 한다.

### 4.3 경쟁 제품군 조사

#### 1. Schneider Electric

- `PowerTag`는 회로 차단기 단위에 붙는 무선 에너지 센서와 게이트웨이 구조를 제공한다.
- 공식 자료에서는 `incoming circuit부터 final electrical load까지` 에너지 사용을 정밀하게 보는 구조를 강조한다.

시사점:

- 글로벌 경쟁사는 분기회로별 데이터 가시성을 매우 중요하게 본다.
- 단순 총전력 모니터링이 아니라 `최종 부하 단위 데이터`가 핵심 경쟁 포인트다.

#### 2. Siemens

- `Sentron Powermind`는 저압 차단기 상태, 전류, 전력, 에너지뿐 아니라 `breaker condition`과 `remaining lifetime`까지 보는 방향을 제시한다.
- 즉 계측을 넘어 `상태기반 유지보수` 관점으로 확장하고 있다.

시사점:

- 시장은 이미 `측정`에서 `예지보전/상태감시` 방향으로 이동 중이다.
- 우리도 1차 제품은 계측+차단에 집중하되, 2차 로드맵에서는 `이력 기반 상태 판단`을 준비해야 한다.

#### 3. Eaton

- `Power Xpert Branch Circuit Monitor`는 실시간 에너지 데이터 원격 확인, 데이터 연계, 과부하 임계치 경보, 다운타임 최소화, 에너지 코드 대응을 강조한다.
- `branch circuit` 단위 감시와 경보, 운영 중단 최소화가 핵심 가치다.

시사점:

- 시장은 `전력 모니터링 정확도`만이 아니라 `운영중단 예방`과 `규정 대응`을 동시에 판다.
- Smart Load도 `절감 + 보호 + 운영 안정성`을 함께 이야기해야 한다.

#### 4. Leviton

- `VerifEye Series 7000`은 최대 48회로를 감시하고 RS-485/Ethernet, Modbus RTU/TCP, BACnet MS/TP/IP를 지원한다.
- 즉 기존 분전반/기존 건물에도 붙일 수 있는 `레트로핏형 분기회로 모니터링`이 중요한 상품 축이다.

시사점:

- 현장 적용성, 기존 설비 호환성, 설치 난이도는 실제 수주 경쟁에서 매우 중요하다.
- Smart Load는 `기존 DCU/PLC/MQTT 구조에 들어가기 쉬운 장치`라는 점을 강점으로 잡아야 한다.

### 4.4 경쟁 제품군 대비 빈틈

현재 조사 기준으로 글로벌 제품군은 아래 두 가지가 강하다.

- 고급 소프트웨어와 통합된 대기업형 솔루션
- 분기회로별 계측 특화형 모니터링 장치

반면 국내 현장 관점에서는 아래 빈틈이 있다.

- 현장 맞춤형 커스터마이징 속도
- DCU/PLC 기반 기존 구조와의 유연한 연동
- 현장 요구에 맞춘 차단 정책, 복구 정책, 주기 정책 조정
- 중소 현장용 도입비용 및 PoC 속도

따라서 우리 포지션은 `글로벌 EMS 축소판`이 아니라 `국내 현장 맞춤형 Smart Load 단말 + SEMS 연동 패키지`가 더 적합하다.

## 5. 우리 제품의 특화점

### 5.1 제품 정체성

우리 제품은 단순 계량기가 아니라 아래 기능을 한 장치에 모으는 것이 핵심이다.

- 분기회로 단위 직접 계측
- 장치 내부 즉시 판단
- 현장 차단 수행
- DCU와의 경량 통신
- 서버 분석용 원천 데이터 제공

### 5.2 특화 포인트

#### 특화점 1. SEMS 말단 실행장치 포지션

대부분 시장 제품은 `측정` 또는 `모니터링`에 강점이 있으나, 우리는 `측정 + 자체 판단 + 차단`을 하나로 묶는다.

#### 특화점 2. 기존 시스템 친화성

- 내부 회의 기준 구조는 이미 `MQTT - 3상 점검장치 - PLC - DCU - Smart Load`로 잡혀 있다.
- 즉 Smart Load는 기존 시스템을 다 바꾸는 제품이 아니라 `기존 구조에 붙는 실행 가능한 말단장치`로 설명할 수 있다.

#### 특화점 3. 1차/2차 확장 로드맵이 명확함

- 1차: 단상, 필수 계측, 과전류/과전압, 차단, RS-485/DCU 연동
- 2차: 이벤트 로그 고도화, 서버 분석 필드 정교화, 부하 패턴 추적
- 3차: ZCT, 누설전류, 고조파, 아크, 상태진단

즉 처음부터 과도한 범위를 약속하지 않고, 되는 것부터 시장에 넣는 로드맵을 제시할 수 있다.

#### 특화점 4. 현장형 커스터마이징

- 복구정책
- 전송주기
- 이벤트 정책
- 차단 임계치
- 상위 시스템 인터페이스

이 다섯 가지는 현장별 요구차가 큰데, 우리 구조는 이를 PM/현장 요구에 맞춰 조정하기 좋다.

### 5.3 제안 메시지

대외 제안에서는 아래 문장을 기본 메시지로 사용한다.

`Smart Load는 SEMS의 말단에서 전력 데이터를 직접 만들고, 이상상황을 현장에서 즉시 판단해 차단까지 수행하며, 상위 DCU/서버가 에너지절감 운영과 분석을 할 수 있도록 원천데이터를 공급하는 실행형 현장단말이다.`

## 6. 제안서 초안

### 6.1 고객 문제 정의

- 분기회로 단위 에너지 사용량이 보이지 않는다.
- 과부하/이상전압 등 현장 위험을 말단에서 바로 잡기 어렵다.
- 상위 서버 분석용 데이터 구조가 부족하다.
- 기존 설비를 모두 교체하지 않고도 SEMS를 단계적으로 도입하고 싶다.

### 6.2 제안 개념

우리는 `Smart Load + DCU 연동 + 상위 SEMS 데이터 구조`를 함께 제안한다.

구성 개념:

1. Smart Load가 분기회로 전력과 상태를 직접 계측
2. 장치가 과전류/과전압/대기전력 정책을 현장에서 수행
3. DCU가 여러 Smart Load를 수집
4. 상위 서버가 누적/이력/패턴/알람을 관리

### 6.3 1차 제안 범위

- 단상 Smart Load
- 전압, 전류, 주파수, 유효전력, 전력량
- 과전류/과전압 차단
- RS-485 기반 DCU 연동
- 상태/이벤트/누적값 전송
- 데모 및 시험 시제품 구축

### 6.4 2차 제안 범위

- 피상전력/역률/무효전력 등 고도화 데이터
- 이력기반 경보
- 복구 정책 세분화
- ZCT 기반 누설전류 확장
- 서버 대시보드/분석 로직 연계

### 6.5 제안 포지셔닝

우리는 아래 세 가지를 동시에 파는 구조가 맞다.

- 현장 안전
- 에너지 절감 운영 데이터
- 향후 확장 가능한 SEMS 인프라

## 8. 업무분장 초안

### 8.1 역할 기준

| 구분 | 주요 역할 |
|---|---|
| PM | 범위 확정, 일정관리, 이슈관리, 대외 커뮤니케이션, 의사결정 정리 |
| 사업/영업 | 타겟 고객 정의, 제안 포인트 정리, 현장 요구 수집 |
| HW | 회로/전원/차단부/보드 설계, 부품선정, 시제품 제작 |
| FW | 계측칩 드라이버, 보호로직, 복구로직, RS-485/DCU 연동 |
| 서버/DCU | 수집 구조, 데이터 필드, 폴링 정책, 이벤트 처리 |
| 품질/시험 | 계측 정확도, 차단/복구, 통신 안정성 시험 |

| 상위 서버/플랫폼 | 조중협 이사 |

## 10. 문서 버전관리 기준

### 10.1 운영 원칙

- 이 문서를 `SEMS/Smart Load 공유용 단일 기준 문서`로 운영한다.
- 발표용 PPT는 이 문서를 요약한 파생 산출물로 본다.
- 주제별 세부 검토 문서는 계속 존재할 수 있으나, 대외 공유/내부 의사결정 기준은 본 문서에 최종 반영한다.

### 10.2 버전 규칙

- `v0.x`: 초안, 방향 검토 단계
- `v1.x`: 대내 공유 기준 확정본
- `v2.x`: 대외 제안/영업 활용 가능본

### 10.3 변경 이력

| 버전 | 일자 | 작성/수정 | 변경 내용 |
|---|---|---|---|
| v0.1 | 2026-04-10 | Codex | 폴더 학습 결과, 시장조사, 특화점, 제안서, WBS, 업무분장, 버전관리 초안 통합 |

### 10.4 변경 승인 규칙

- 시장 포지셔닝 변경: PM 승인
- 1차 제품 범위 변경: PM + HW/FW 합의
- 대외 메시지 변경: PM + 사업 검토
- 일정 마일스톤 변경: PM 승인 후 변경 이력 반영
