## 2. 권장 작업 순서

### 2.1 요구사항 정리

먼저 아래를 확정해야 합니다.

- 단상인지, 향후 다채널 확장인지
- 차단 목적이 과전류 중심인지, 누전/아크까지 포함하는지
- 필수 표시 항목과 서버 전송 항목이 같은지 다른지
- 통신 프로토콜이 실제로 `RS-485 + DLMS/COSEM`인지
- 인증 대상이 `누전차단`, `절연열화 감시`, `전력량 계측` 중 어디까지인지

이 단계에서 가장 중요한 판단은 다음입니다.

- `BL0942 단독 계량기`로 갈지
- `BL0942 + 별도 ZCT 계측 회로 + MCU 계산`으로 갈지
- `처음부터 다른 계량 SoC/AFE`로 갈지

### 2.2 하드웨어 구조 확정

BL0942 기준으로는 아래가 먼저 정해져야 합니다.

- 전압 센싱 방식: 저항 분압
- 부하 전류 센싱 방식: 션트 또는 CT 전처리 방식
- 릴레이/차단기 구동 방식
- RS-485 트랜시버 선정
- 보조전원 구조
- 절연 및 안전 규격

ZCT를 넣을 계획이면 추가로 아래가 필요합니다.

- ZCT 센서 사양 선정
- ZCT 아날로그 프론트엔드 설계
- MCU ADC 또는 별도 AFE 입력 경로
- 위상 기준 전압과의 동기 샘플링 구조
- IGR/IGC 분리 알고리즘

### 2.3 펌웨어 구조 설계

펌웨어는 아래 순서로 나누는 게 좋습니다.

1. 보드 초기화
2. BL0942 드라이버
3. 계측값 스케일 변환
4. 통계값 계산
5. 보호 이벤트 판단
6. 차단기 제어
7. 통신 프레임 처리
8. 저장/로그/시각 동기

권장 모듈 구조:

- `drv_bl0942`: 레지스터 읽기/쓰기, CRC/UART 또는 SPI
- `app_metering`: V/I/W/Wh/Freq 변환
- `app_power_quality`: PF, 피상전력 등 파생값 계산
- `app_protection`: 과전류/과전압/부족전압 판단
- `app_leakage`: ZCT 입력 처리, IGR/IGC 계산
- `app_breaker`: 트립/재폐로/상태 확인
- `app_comm`: RS-485 DLMS 또는 현장 프로토콜
- `app_log`: 이벤트/통계 저장

### 2.4 시험 및 검증

최소 시험 항목은 아래와 같습니다.

- 전압 RMS 정확도
- 전류 RMS 정확도
- 유효전력 정확도
- 전력량 누적 정확도
- 주파수 측정 정확도
- 역률 계산 오차
- 과전류 검출 응답시간
- 차단기 릴레이 동작 시간
- 전원 순간강하/복전 복구
- 통신 장애 복구

ZCT 포함 시 추가 시험:

- 누설전류 정확도
- IGR/IGC 분리 정확도
- 누설 이벤트 응답시간
- 온습도/노이즈 환경에서 오검출 여부
- 인증 기준치 만족 여부

## 3. `sems_측정항목.docx` 항목 분류

아래 분류는 문서 원문을 기준으로 한 실무용 재정리입니다.

### 3.1 직접 계측 항목

센서/칩에서 직접 읽어오는 값입니다.

- 전압
- 주파수
- 부하 전류
- 유효전력
- 제품 내부온도
- 누전차단용 영상전류
- 저항성 누설전류
- 용량성 누설전류
- 아크전류

주의:

- `제품 내부온도`는 BL0942 계량 기능이 아니라 별도 온도센서 또는 MCU 내부 온도센서 영역입니다.
- `영상전류`, `저항성/용량성 누설전류`, `아크전류`는 BL0942 기본 계측 범위가 아닙니다.

### 3.2 계측값으로 계산하는 항목

직접 계측값을 기반으로 MCU 또는 계량칩 결과로 계산하는 값입니다.

- 전력량
- 과전류 판단
- 무효전력
- 피상전력
- 역율(%)
- 고조파 발생현황

설명:

- `전력량`은 유효전력을 시간 적분한 값입니다.
- `과전류`는 전류 측정값과 설정 임계치 비교 결과입니다.
- `무효전력`, `피상전력`, `역율`은 전압/전류/위상 정보로부터 계산되는 파생값입니다.
- `고조파 발생현황`은 파형 또는 주파수 성분 분석이 필요합니다.

### 3.3 추후 ZCT 적용 시 작업 항목

문서 내용 중 ZCT가 들어가야 의미 있게 구현 가능한 항목입니다.

- 누전차단용 영상전류 측정
- 저항성 누설전류(IGR) 계산
- 용량성 누설전류(IGC) 계산
- 누설전류 임계치 초과 판단
- 누설전류 최대/최소/평균 통계
- 누설 이벤트 발생 시각/지속시간 기록
- 인증 기준에 맞는 정확도 검증

추가 실무 작업:

- ZCT 선정
- ZCT 신호 레벨링/증폭/필터 회로 설계
- 기준 전압 대비 위상 계산 방식 정의
- IGR/IGC 분리 수식 검증
- 누설차단 임계값 정책 확정
- 인증 시험 절차 정리

## 4. BL0942로 가능한 것과 어려운 것

판정 기준:

- `직접 가능`: BL0942 레지스터/기능으로 직접 취득 가능
- `간접 가능`: BL0942 출력값을 MCU가 추가 계산하면 가능
- `사실상 어려움`: BL0942만으로는 부족하고 추가 하드웨어 또는 다른 칩이 필요

### 4.1 가능/불가 판정표

| 항목 | 구분 | BL0942 판정 | 메모 |
|---|---|---|---|
| 전압 RMS | 직접 계측 | 직접 가능 | 지원 |
| 부하 전류 RMS | 직접 계측 | 직접 가능 | 지원 |
| 유효전력 | 직접 계측 | 직접 가능 | 지원 |
| 전력량 | 계산/누적 | 직접 가능 | 에너지 펄스/카운터 지원 |
| 주파수 | 직접 계측 | 직접 가능 | FREQ 레지스터 지원 |
| 과전류 | 보호 판단 | 직접 가능 | Fast RMS + 임계치 가능 |
| 역율(PF) | 계산 | 간접 가능 | V, I, W 기반 MCU 계산 가능 |
| 피상전력 | 계산 | 간접 가능 | S = V x I |
| 무효전력 | 계산 | 조건부 간접 가능 | PF 또는 위상 정보 기반 추정은 가능하나 정확도/정의 확인 필요 |
| 고조파 발생현황 | 분석 | 사실상 어려움 | 파형 샘플은 있으나 하모닉 엔진 없음 |
| 영상전류(ZCT) | 직접 계측 | 사실상 어려움 | 별도 누설전류 입력 채널 없음 |
| 저항성 누설전류(IGR) | 계산 | 사실상 어려움 | ZCT + 위상분해 추가 필요 |
| 용량성 누설전류(IGC) | 계산 | 사실상 어려움 | ZCT + 위상분해 추가 필요 |
| 아크전류 | 분석/보호 | 사실상 어려움 | 고주파/아크 특화 검출 구조 필요 |
| 제품 내부온도 | 직접 계측 | 불가 | 별도 센서 필요 |
| MAC/IP Address 포함 전송 | 통신 | 칩 무관 | MCU/통신모듈 영역 |

### 4.2 BL0942 공식 자료 기준으로 확인된 항목

공식 자료상 BL0942는 아래 기능이 명시되어 있습니다.

- 전류/전압 RMS 측정
- 유효전력 측정
- 유효전력량 측정
- 빠른 전류 RMS
- 과전류 검출
- 전압/전류 영점 통과 출력
- 샘플 파형 데이터 제공
- 주파수 레지스터 제공

즉, BL0942는 `단상 1전압 + 1전류` 계량칩으로 보는 것이 맞고, 주된 강점은 `전압/전류/유효전력/전력량/주파수/과전류`입니다.

반대로 아래는 공식 핵심 기능으로 보이지 않습니다.

- 전용 누설전류(ZCT) 측정 채널
- IGR/IGC 직접 분리
- 고조파 THD/차수별 분석
- 아크 fault 전용 검출
- 내부 온도 측정
- 무효전력/피상전력/역률 직접 레지스터

## 5. 가장 중요한 설계 판단

### 5.1 BL0942 단독 사용 시 현실적인 범위

BL0942만 쓰면 우선순위는 아래처럼 잡는 것이 현실적입니다.

- 전압
- 전류
- 유효전력
- 전력량
- 주파수
- 과전류 차단
- 차단기 상태
- 통신 전송
- PF/피상전력 정도의 파생 계산

즉, `스마트 전력 차단기`는 가능하지만, `누전/절연열화/아크 진단까지 포함한 고급 보호형 지선차단기`를 BL0942 단독으로 처리하기는 어렵습니다.

### 5.2 ZCT를 꼭 넣어야 한다면

다음 구조가 필요합니다.

- BL0942: 부하 전압/부하 전류/유효전력/전력량 담당
- ZCT + 별도 아날로그 회로: 누설전류 검출 담당
- MCU ADC 또는 별도 AFE: ZCT 파형 샘플링
- MCU 알고리즘: IGR/IGC 분리, 누설 이벤트 판단

이 경우 펌웨어는 최소 2개 계측 경로를 합쳐야 합니다.

- 전력 계측 경로
- 누설 계측 경로

### 5.3 문서 간 충돌 사항

기능명세 문서는 `MAX71315` 기반이라 아래 전제가 깔려 있습니다.

- 계량 SoC 내부 기능이 더 넓다
- ZCT 채널 연동을 염두에 두고 있다
- OBIS/DLMS 중심으로 설계되어 있다

하지만 BL0942로 바꾸면 아래 작업이 다시 필요합니다.

- 레지스터 맵 재정의
- 통신 드라이버 재작성
- 계측/보호 가능 범위 재정의
- ZCT 측정 경로 별도 설계
- 정밀도/인증 가능성 재검토

즉, `기능명세 v0.2를 그대로 구현`이 아니라 `BL0942 버전 기능명세 v0.1`을 새로 만드는 것이 맞습니다.
