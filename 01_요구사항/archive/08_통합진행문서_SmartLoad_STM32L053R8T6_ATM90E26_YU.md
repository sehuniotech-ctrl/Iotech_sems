기존처럼 칩 기능, 구매성, 일정, 데모 테스트, 하드웨어 전달, 펌웨어 개발이 여러 문서에 흩어지지 않도록 아래 순서대로 정리한다.

2. 가격 / Delivery / 구매성 정리
3. 계량칩 관련 일정 관리
4. 기존 eval 보드와 `계량칩 + SSOP 보드`로 데모 테스트 선실행
5. 기능 및 핀 선정 후 HW팀 전달
6. 4번 기준 코드 개발 및 데모 테스트
7. `STM32L053R8T6 + ATM90E26-YU` 보드 기준 코드 개발 및 데모 테스트
8. 기능 추가

현재 기준 칩 조합은 아래와 같다.

- MCU: `STM32L053R8T6`
- 계량칩: `ATM90E26-YU`

## 2. Smart Load 기능 구현 가능 여부 비교표

아래 표는 Smart Load에서 요구하는 기능을 `STM32L053R8T6 + ATM90E26-YU` 조합으로 구현 가능한지 정리한 것이다.  
주의할 점은 계측 기능은 주로 `ATM90E26-YU`가 담당하고, `STM32L053R8T6`은 제어 / 통신 / 판단 / 로그 / 차단 로직을 담당한다는 점이다.

| 항목 | STM32L053R8T6 기준 구현 여부 | 구현 방식 | 비고 |
|---|---|---|---|
| 전압 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 주파수 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 부하 전류 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 전력량 | 가능 | 계량칩 값 수신 + MCU 누적 관리 | 유효전력량 기준 |
| 과전류 | 가능 | MCU 임계값 비교 + 릴레이 제어 | 보호 로직 필요 |
| 유효전력 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 무효전력 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 피상전력 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 역률 | 가능 | `ATM90E26-YU` 계측값 수신 및 전송 | 계량칩 직접 계측 |
| 과전압 차단 | 가능 | MCU 임계값 비교 + 릴레이 제어 | 보호 로직 필요 |
| RS-485 통신 | 가능 | UART + 외부 RS-485 트랜시버 | DCU polling 대응 |
| 날짜/시간 포함 전송 | 가능 | MCU 처리 | RTC 또는 상위 시간 동기화 필요 |
| MAC/IP 포함 전송 | 가능 | MCU/통신부 처리 | 시스템 구조 기준 |
| 이벤트 로그 | 가능 | MCU 저장 및 상위 전송 | 정책 필요 |
| 대기전력 차단 | 가능 | 전력값 판단 + 릴레이 제어 | 정책 확정 필요 |
| 대기전력 복구 | 가능 | 스케줄 / 원격 명령 기반 | polling 구조 기준 |
| 누전차단 | 현재 1차 불가 | 별도 센서/회로 필요 | ZCT 필요 |
| 영상전류 | 현재 1차 불가 | 별도 ZCT 경로 필요 | 계량칩 단독 불가 |
| 저항성 누설전류 | 현재 1차 불가 | ZCT + 위상분석 필요 | 2차 확장 |
| 용량성 누설전류 | 현재 1차 불가 | ZCT + 위상분석 필요 | 2차 확장 |
| 고조파 발생현황 | 현재 1차 불가 | 별도 분석 경로 필요 | 2차 확장 |
| 아크전류 | 현재 1차 불가 | 별도 회로/알고리즘 필요 | 2차 확장 |
| 제품 내부온도 | 조건부 가능 | 별도 온도센서 또는 MCU 내부센서 | 계량칩 직접 기능 아님 |

### 2.1 해석

- 현재 `1차 제품 범위`는 `STM32L053R8T6 + ATM90E26-YU`로 충분히 구현 가능하다.
- 다만 `누전`, `영상전류`, `Igr/Igc`, `고조파`, `아크`는 MCU 문제가 아니라 별도 하드웨어 경로가 필요한 항목이므로 현재 1차 범위에서는 제외한다.
- 따라서 현재 칩 조합은 `단상 Smart Load 1차 개발용`으로는 적합하다.

## 3. 가격 / Delivery / 구매성 정리 표

### 3.1 최종 선정 칩 기준

| 구분 | 부품명 | 단가 | Delivery | MOQ | 공급성 / 비고 |
|---|---|---:|---|---:|---|
| MCU | `STM32L053R8T6` | `$1.10` | `ASAP` | `960 pcs` | 인셈 회신 기준, Buffer 보유 |
| 계량칩 | `ATM90E26-YU-B` | 약 `$1.62` (1pcs), `$1.35` (100pcs) | 약 `7 Weeks` | 유통처 기준 | DigiKey 기준 |
| 계량칩 | `ATM90E26-YU-R` | 약 `$1.66` (1pcs), `$1.39` (100pcs) | 약 `4~7 Weeks` | 유통처 기준 | Mouser/Arrow 기준 |

### 3.2 후보 MCU 비교 관점

| 항목 | STM32L053R8T6 | STM32U073RCT6 |
|---|---|---|
| 기능 구현 가능 여부 | 가능 | 가능 |
| 저전력 특성 | 유리 | 더 유리 |
| 가격 | `$1.10` | 소폭 더 유리 |
| Delivery | `ASAP` | `20 Weeks` |
| 조달 리스크 | 낮음 | 높음 |
| 최종 판단 | 선정 | 제외 |

### 3.3 판단 요약

- `STM32U073RCT6`는 저전력과 성능 면에서는 좋은 후보였으나, 납기 `20 Weeks`가 가장 큰 리스크였다.
- `STM32L053R8T6`는 Delivery `ASAP`와 Buffer 보유 조건으로 현재 프로젝트 일정에 더 적합하다.
- 따라서 최종 MCU는 `STM32L053R8T6`로 간다.

## 5. 기존 eval 보드 + 계량칩 SSOP 보드 기준 데모 테스트

### 5.1 목적

최종 MCU 보드가 나오기 전에 `기존 eval 보드`와 `ATM90E26-YU SSOP 보드`를 이용해 계량칩이 정상 동작하는지 먼저 확인한다.

### 5.2 이번 단계에서 확인할 것

- `ATM90E26-YU` 통신 가능 여부
- 초기화 가능 여부
- 전압 / 전류 / 전력 / 주파수 읽기 가능 여부
- 계량값이 정상적으로 변화하는지
- 계량칩 기본 레지스터 read/write 확인

### 5.3 산출물

- 데모 테스트 결과 정리
- 초기 계량값 로그
- 펌웨어 통신 드라이버 초안

### 5.4 현재 판단

이 단계는 `최종 보드가 없더라도 먼저 진행 가능한 단계`이므로 가장 먼저 착수하는 것이 맞다.

## 11. 결론

현재 Smart Load 개발은 `STM32L053R8T6 + ATM90E26-YU` 기준으로 진행한다.  
핵심은 바로 최종 보드 개발로 들어가기보다, 먼저 `기존 eval 보드 + 계량칩 SSOP 보드`로 데모 테스트를 실행해 계량칩과 코드 구조를 선검증하는 것이다.  
이후 기능 및 핀을 확정해 HW팀에 전달하고, 최종 보드 기준 코드 개발로 이어가는 순서가 가장 안전하다.
