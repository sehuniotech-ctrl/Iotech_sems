# 기능 구분

작성일: 2026-04-03  
기준 조합: `BL0942 + STM32G071CBT6`

## 1. 목적

지선차단기에서 직접 계측하거나 생성해야 하는 값과, 서버에서 받아서 계산/분석해야 하는 값을 구분한다.

## 2. 구분 원칙

- `지선차단기의 계측값`: 현장 디바이스가 직접 측정하거나 즉시 판단해서 전송하는 값
- `서버에서 계산할 값`: 여러 시점/여러 장치 데이터를 모아 집계, 분석, 비교, 추론하는 값

## 3. 지선차단기의 계측값

### 3.1 직접 계측값

- 전압 RMS
- 전류 RMS
- 유효전력
- 유효전력량
- 주파수
- 빠른 전류 RMS

### 3.2 지선차단기 내부 판단값

- 과전류 여부
- 과전압 여부
- 차단 여부
- 차단기 상태
- 이벤트 발생 여부
- 이벤트 발생 시각

### 3.3 지선차단기가 계산해서 보내는 값

- 피상전력
- 역률
- 전압 최대값
- 전압 최소값
- 전압 평균값
- 전류 최대값
- 전류 최소값
- 전류 평균값

## 4. 서버에서 계산/분석할 값

### 4.1 집계값

- 시간대별 사용전력 집계
- 일/주/월 누적 전력량
- 지점별 전력 사용 비교
- 지선별 전력 사용 순위

### 4.2 분석값

- 부하 패턴 분석
- 이상 사용 패턴 탐지
- 시간대별 피크 분석
- 경향 분석
- 이벤트 발생 빈도 분석

### 4.3 절감/운영 관련 값

- 절감 전후 비교값
- 목표 대비 절감율
- 운영 효율 지표
- 설비별 사용 특성 분석

### 4.4 향후 확장 분석값

- 카메라/열화상 연계 분석값
- 부스바 온도와 전력 사용 상관분석
- 경보 우선순위 판단

## 5. 이번 1차 제품 기준 정리

### 5.1 지선차단기에서 반드시 해야 하는 것

- 계측
- 보호 판단
- 차단
- 상태 전송
- 이벤트 전송

### 5.2 서버에서 해야 하는 것

- 장기간 누적 분석
- 여러 장치 비교
- 절감 효과 계산
- 시각화
- 리포트 생성

## 6. 정리

즉, 이번 구조는 아래처럼 정리한다.

- 지선차단기: `측정 + 즉시 판단 + 차단 + 전송`
- 서버: `누적 + 비교 + 분석 + 절감 계산 + 리포트`
